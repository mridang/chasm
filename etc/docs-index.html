chasm

OpenAPI 3 mock server in Rust. API documentation for the workspace.

chasm is a workspace of three crates that together implement an OAS3 mock server. You give it an OpenAPI 3 spec, you get back mock responses — either as a standalone HTTP server, or as a library you embed.

Crates

chasm-engine

The mock engine itself. Spec + request → response. No HTTP layer. This is what you embed if you're building a custom harness, a Lambda handler, or a WASM bundle. Start here if you want the library API.

 chasm-faker

Standalone JSON Schema → fake value generator. Pure schema-to-value; no OpenAPI awareness. chasm-engine calls this internally when a spec has no example: declared, but you can use it directly to generate test data from any JSON Schema.

 chasm-server

The Axum HTTP binary that wraps chasm-engine. Runs as chasm-server spec.yaml --port 4010. This crate is documented because it's part of the workspace, but you usually consume it as a binary, not a library.

Quick start (library)

use chasm_engine::{load_spec, mock, MockConfig, MockRequest};

let spec = load_spec(include_str!("petstore.yaml")).unwrap();

let mut req = MockRequest::default();
req.method = "GET".into();
req.path   = "/pets".into();

let resp = mock(&spec, &req, &MockConfig::default()).unwrap();

assert_eq!(resp.status, 200);
println!("{}", serde_json::to_string_pretty(&resp.body).unwrap());

Quick start (server)

chasm-server petstore.yaml --port 4010
curl http://localhost:4010/pets
